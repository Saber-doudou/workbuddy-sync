## 1. 配置向导 Prompt

### 1.1 初始化配置

```
你是一个 WorkBuddy Sync 配置向导。请帮助用户完成以下配置：

1. 检测用户的 WorkBuddy 配置目录
2. 询问用户需要备份的内容类别：
   - 核心配置文件（SOUL.md, USER.md 等）
   - 技能和插件配置
   - 记忆文件
   - 自动化配置
   - 连接器配置
   - 工作空间配置
   - 知识内容
   - IMA 凭证
   - 微云备份

3. 根据用户选择生成 config.user.py 配置文件
4. 提供下一步操作指引

请使用友好的语气，逐步引导用户完成配置。
```

### 1.2 微云配置

```
用户想要启用微云备份。请帮助用户完成以下配置：

1. 确认用户已安装微云连接器
2. 获取微云连接器目录路径
3. 获取微云脚本目录路径
4. 获取微云目录 Key（pdir_key）
5. 验证配置是否正确
6. 测试微云连接

如果配置失败，请提供故障排除建议。
```

## 2. 同步任务 Prompt

### 2.1 日常同步

```
用户想要同步 WorkBuddy 配置到云端。请执行以下步骤：

1. 检查 IMA 凭证是否存在
2. 运行健康检查确认配置完整性
3. 执行同步命令：python sync_to_cloud.py
4. 解析同步结果：
   - SUCCESS: 同步成功
   - UNCHANGED: 配置无变更
   - PARTIAL: 部分成功（IMA 成功，微云失败）
   - FAILED: 同步失败

5. 根据结果提供反馈：
   - 成功：显示同步统计
   - 失败：提供故障排除建议
   - 无变更：提示用户无需操作

请确保提供清晰的状态反馈。
```

### 2.2 强制同步

```
用户想要强制同步配置（跳过变更检测）。请执行以下步骤：

1. 确认用户确实需要强制同步
2. 警告用户强制同步会覆盖云端备份
3. 执行强制同步：python sync_to_cloud.py --force
4. 显示同步结果

请确保用户了解强制同步的影响。
```

## 3. 健康检查 Prompt

### 3.1 完整健康检查

```
用户想要检查 WorkBuddy Sync 配置是否正确。请执行以下检查：

1. 检查 WorkBuddy 配置目录是否存在
2. 检查工作空间目录是否存在
3. 检查 IMA 凭证是否配置
4. 检查核心配置文件是否存在
5. 检查备份类别配置
6. 检查微云配置（如果启用）
7. 检查指纹文件
8. 检查槽位映射

运行命令：python sync_to_cloud.py --health-check

根据检查结果：
- 全部通过：提示用户可以开始同步
- 有警告：列出警告项并提供建议
- 有错误：列出错误项并提供解决方案
```

### 3.2 快速诊断

```
用户遇到同步问题。请快速诊断以下常见问题：

1. IMA 凭证是否过期？
2. 网络连接是否正常？
3. 微云连接器是否配置正确？
4. 工作空间目录是否存在？
5. 磁盘空间是否充足？

请提供简洁的诊断结果和解决方案。
```

## 4. 故障排除 Prompt

### 4.1 同步失败

```
用户报告同步失败。请按以下步骤排查：

1. 查看错误消息
2. 检查错误类型：
   - FAILED: 通用失败
   - FATAL_ERROR: 脚本异常
   - NO_DATA: 无数据

3. 常见原因及解决方案：
   - IMA 凭证过期：重新获取凭证
   - 网络问题：检查网络连接
   - 磁盘空间不足：清理磁盘空间
   - 配置文件损坏：重新生成配置

4. 提供详细的操作步骤

请确保用户能够按照步骤解决问题。
```

### 4.2 微云上传失败

```
用户报告微云上传失败。请按以下步骤排查：

1. 检查微云连接器是否配置正确
2. 检查微云目录 Key 是否正确
3. 检查网络连接
4. 检查微云存储空间
5. 查看错误日志

常见解决方案：
- 重新配置微云连接器
- 更新微云目录 Key
- 清理微云存储空间
- 等待网络恢复后重试

请提供清晰的排查步骤。
```

## 📝 使用说明

### 如何使用这些 prompt

1. **复制粘贴**：直接复制需要的 prompt 到 AI 助手
2. **自定义修改**：根据具体需求修改 prompt 内容
3. **组合使用**：将多个 prompt 组合成完整的工作流

### Prompt 设计原则

1. **清晰明确**：使用简洁明了的语言
2. **步骤清晰**：将复杂任务分解为小步骤
3. **错误处理**：考虑可能的错误情况
4. **用户友好**：使用友好的语气和格式

### 最佳实践

1. **上下文信息**：提供足够的上下文信息
2. **具体要求**：明确具体的要求和期望
3. **示例说明**：提供示例说明
4. **反馈机制**：建立清晰的反馈机制
